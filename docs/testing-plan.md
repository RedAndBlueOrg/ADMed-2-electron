테스팅 플랜
==========

사전 준비
--------
- `.env`에 유효한 API 주소를 넣고 `npm install && npm start`로 기동.  
- `%TEMP%/admed-cache`를 확인해 캐시가 쌓이는지 모니터링.  
- `device_config.ini`에 테스트용 기기 시리얼을 설정하거나 기본값으로 진행.

기능별 수동 테스트
---------------
- **재생목록 로드**: 네트워크 정상/오프라인 상태에서 시나리오 요청, `templates` 필드 없는 경우 에러 오버레이가 뜨는지 확인.  
- **캐시/다운로드**: 최초 실행 시 원격 스트림으로 재생 후 캐시 파일이 생성되는지, 2회차 실행에서 로컬 파일로 곧바로 재생되는지 확인.  
- **HLS ZIP**: ZIP 형태 템플릿을 내려 테스트 → 압축 해제 후 `.m3u8`가 로컬 URL로 재생되는지, 캐시 정리가 잘 되는지 확인.  
- **영상/이미지 전환**: 이미지 `time` 값에 따라 정해진 시간 후 다음 항목으로 넘어가는지, 영상 `ended` 이벤트로 전환되는지 확인.  
- **공지 바**: 공지 리스트가 존재할 때만 롤링되고, 텍스트 길이에 따라 속도가 조절되는지 확인.  
- **대기열/클리닉**: `waitingInfo = 'Y'`, `memberSeq`가 있는 시나리오로 초기 목록/실시간 WS 메시지 수신을 검증. 새 호출 시 알림·효과음·TTS 발생 여부 확인.  
- **날씨 패널**: `WEATHER_LAT/LON` 설정 시 정상 표시, 미설정 시 위치 권한 요청 또는 IP 기반 Fallback 동작 확인.  
- **오프라인 복구**: 네트워크 차단 후 에러 오버레이 표시 → 온라인 복귀 시 자동 재시도 및 WS 재연결 확인.  
- **창/운영 기능**: 컨텍스트 메뉴(창 크기 프리셋, 위치 복원, 전체화면/항상 위, 새로고침)와 트레이 동작, “닫기” 후 트레이 최소화, “종료” 동작 확인.  
- **관리자 설정**: 컨텍스트 메뉴 → 관리자 설정 → 비밀번호 입력 → 시리얼 변경 후 시나리오가 새로고침되는지 확인.  
- **자동 실행**: 자동 실행 토글 후 OS 재부팅 시 앱이 함께 시작되는지 확인.
- **랜딩 페이지**: 시리얼을 비운 상태에서 실행하면 랜딩 페이지(`LANDING_URL`)가 표시되고, 시리얼을 설정하면 정상 재생으로 전환되는지 확인.

성능/안정성 체크
--------------
- 장시간 재생(>4시간) 동안 메모리/CPU 사용량 추적, HLS 버퍼링/멈춤 현상 없는지 모니터링.  
- 캐시 정리 로직이 대용량 ZIP/다수 파일에서 시간 초과 없이 동작하는지 확인.  
- WebSocket 연결 불안정 시 재연결 간격(지수 백오프)이 과도하지 않은지 확인.
